# 9. Backlog do Produto

## **9.1 Backlog Geral com Priorização e MVP**
| ID | Nome | História de Usuário | Prioridade | MVP |
| :---- | :---- | :---- | :---- | :---- |
| US01 | Gerenciamento de Organizações | Como **Administrador**, quero cadastrar, editar e remover organizações, para centralizar leads, contatos e negócios.  | Must | x |
| US02 | Gerenciamento de Usuários | Como **Proprietário**, quero atribuir, editar e remover quaisquer usuários de organizações, para definir quem pode trabalhar com os leads, contatos e negócios da organização. | Should |  |
| US03 | Personalização de Funis de Vendas | Como **Gerente de Vendas**, quero personalizar funis de vendas, para adaptá-los ao processo comercial da empresa. | Could |  |
| US04 | Gerenciamento de Leads | Como **Representante de Desenvolvimento de Vendas**, quero cadastrar, editar e remover manualmente novos leads, para realizar a triagem. | Must | x |
| US05 | Temperatura de Leads | Como **Representante de Desenvolvimento de Vendas** quero ver a temperatura de um lead (quente,morno,frio) para definir seu nível de prioridade de atendimento e quais decisões tomar. | Should |  |
| US06 | Visualização de Leads | Como **Representante de Desenvolvimento de Vendas** quero ver os dados de um lead (Nome, telefone, idade, emprego, interesses, etc…) nas etapas iniciais do funil (ex: Prospecção, Qualificação, Repescagem ou Follow Up) para poder negociar com ele efetivamente.  | Must | x |
| US07 | Visualização de Dashboards | Como **Usuário** quero ver dashboards com taxas de conversão e outros dados dos clientes que minha organização possui. | Could |  |
| US08 | Envio de mensagens para Leads | Como **Representante de Desenvolvimento de Vendas** quero mandar mensagens no Whatsapp para meus leads pelo CRM.  | Should  |  |
| US09 | Gestão de Funil | Como **Coordenador de Vendas**, quero realizar o gerenciamento das etapas dentro do seu funil, podendo mover leads e monitorar o status.  | Must | x |
| US10 | Automatização de informações de Leads | Como **Representante de Desenvolvimento de Vendas**, quero que as informações de origem do usuário (campanha, plataforma e etc) sejam registradas automaticamente para economizar tempo.   | Should |  |
| US11 | Automatização de Confirmação e Follow-up | Como **Representante de Desenvolvimento de Vendas**, quero automatizar mensagens de follow-up e confirmação para reter o lead.  | Could |  |
| US12 | Gerenciamento de Times | Como **Gerente de Vendas**, quero cadastrar, editar e remover times de venda para gerenciar equipes de vendas em diferentes filiais. | Must |  |
| US13 | Atribuição e Monitoramento em Times | Como **Gerente de Vendas**, quero atribuir leads a coordenadores ou membros de sua equipe e monitorar o progresso em cada etapa do funil. | Must |  |
| US14 | Comunicação Interna  | Como **Coordenador de Vendas**, quero ter acesso a ferramentas internas para comunicação com a equipe (comentários nos leads). | Could |  |
| US15 | Mudar o Status do Lead | Como **Representante de Desenvolvimento de Vendas**, quero ter a capacidade de marcar o lead como "qualificado" ou "desqualificado" e mover para a próxima etapa ou arquivar. Também é possível preencher um questionário para que o lead tenha um score dentro da ferramenta. | Must | x |
| US16 | Visualização de Oportunidades | Como **Closer**, quero ter acesso às etapas finais do funil (ex: Proposta, Negociação, Fechamento) para poder fechar negócio com o lead. | Must | x |
| US17 | Marcar negócio como “ganho” ou “perdido” | Como **Closer**, quero marcar o negócio como "ganho" ou "perdido" e registrar o valor da venda para que possa ser registrado e utilizado para decisões futuras. | Must | x |

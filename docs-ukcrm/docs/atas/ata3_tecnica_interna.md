### Deliberações e Assuntos Tratados

**1. Definição da Stack Tecnológica:**

A equipe discutiu e definiu o conjunto de tecnologias que servirá como base para o desenvolvimento do sistema de CRM. A stack tecnológica aprovada é:

- Backend: Django
- Frontend: React
- Banco de Dados: PostgreSQL
- Conteinerização: Docker
- CI/CD (Integração e Entrega Contínua): GitHub Actions
- Comunicação: Desenvolvimento baseado em APIs
- Hospedagem: Hostinger

**2. Metodologia de Desenvolvimento:**

Foi acordado que o projeto seguirá uma metodologia de desenvolvimento Ágil, combinando práticas dos frameworks Scrum e XP (Extreme Programming). O trabalho será organizado em Sprints, com cerimônias como planejamento de Sprint (Planning) e revisões (Review).

Os papéis e as responsabilidades principais da equipe foram distribuídos da seguinte forma:

- Desenvolvedores FullStack: João Guilherme e Marcos Vinícius

- Infraestrutura / DevOps: Pedro Teixeira.
- Designer (UI/UX): Lorena
- Analistas de Requisitos: A responsabilidade será compartilhada por toda a equipe, com participação ativa em todas as fases.


**4. Planejamento Preliminar das Sprints:**

A equipe realizou um planejamento inicial das duas primeiras Sprints para dar início ao desenvolvimento:

Sprint 1: Estruturação e Base do Projeto
- Definição e escrita do Backlog inicial do produto.
- Criação e configuração do ambiente de desenvolvimento com Docker.
- Configuração do ambiente de deploy inicial.
- Elaboração do design inicial das principais telas (wireframes e mockups).

Sprint 2: Funcionalidades Iniciais e MVP
- Desenvolvimento do sistema de cadastramento e autenticação de usuários.
- Implementação da funcionalidade de cadastramento de Leads.
- Criação do protótipo funcional do Funil de Vendas (Kanban).
- Foco em testes de acessibilidade e melhorias na experiência do usuário (UX).
